# Skill: User Flow Design

## Purpose
Map how a specific user type (ICP) moves through the LL website from entry to
exit or conversion action.

## Inputs
- ICP definition (ICP-01 Ontario SMB / ICP-02 Payments/Fintech / other)
- Entry point (organic search / Google Ads / direct / referral)
- Target conversion action (inquiry form / Level 3 page contact / lead magnet)
- Page inventory

## Process
1. Define the entry point for this ICP.
2. Map the likely first page landed on.
3. Define the decision point: what does the user need to understand to continue?
4. Map the next destination (Level 2 routing page or direct to Level 3).
5. Define the exit condition (conversion) or drop-off risk.
6. Identify friction points where users are likely to leave.
7. Recommend UX changes to reduce friction at each step.

## Outputs
- Annotated user flow: entry → pages → decision points → conversion or exit
- Friction point analysis
- Recommended routing improvements

## Constraints
- Flows must respect POL-051 page level roles.
- Level 3 pages are the primary conversion surfaces; do not design Level 2 pages
  to absorb conversion behavior.

## Invocation
Used when designing a new page or auditing conversion performance on an existing
user path.

# Skill: Visual Hierarchy Optimization

## Purpose
Define or audit the visual hierarchy of a page so the user's eye moves in the
intended sequence: primary → secondary → tertiary → action.

## Inputs
- Page content and section inventory
- Page goal (educate / route / convert)
- ICP reading behavior (scanning vs. reading)

## Process
1. Identify the primary focal point of each section.
2. Assign typographic scale (H1/H2/H3, body, caption) to enforce that hierarchy.
3. Define spacing to separate hierarchy levels.
4. Verify: primary is visually dominant; secondary supports; tertiary recedes.
5. If auditing: identify where hierarchy breaks down (competing primaries,
   undersized H1, body text overload, etc.).
6. Recommend corrections.

## Outputs
- Hierarchy map per section (primary / secondary / tertiary elements)
- Typographic scale (H1–H6, body, caption — size / weight / line-height)
- Spacing scale in use
- Audit findings and remediation (if auditing)

## Constraints
- One H1 per page.
- No skipped heading levels.
- Primary focal point must be visually distinct from secondary without relying
  solely on color.

## Invocation
Used during wireframe production and UX audit.

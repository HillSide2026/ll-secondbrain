# Solution Scope: CARF Due Diligence

Component 4 of CARF_PROGRAM. Depends on CARF_SCOPING_AND_CLASSIFICATION.

## Purpose

Defines rule-based triggers for re-assessment of user CARF data.

## Included

- Re-verification trigger rules including:
  - Changes to user information
  - Inconsistencies (e.g., IP location vs declared jurisdiction)
  - Periodic refresh (e.g., every 2–3 years)
- Rule table maintained in policy documentation
- Manual review queue where required

## Excluded

- Initial onboarding data capture (CARF_CLIENT_ONBOARDING)

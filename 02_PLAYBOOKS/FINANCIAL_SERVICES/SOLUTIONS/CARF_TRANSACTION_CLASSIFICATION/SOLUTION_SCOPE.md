# Solution Scope: CARF Transaction Classification

Component 5 of CARF_PROGRAM. Depends on CARF_SCOPING_AND_CLASSIFICATION.

## Purpose

Establishes the system by which the entity consistently determines reportable
transactions and their categorization.

## Minimum Classification Categories

- Crypto-to-fiat
- Crypto-to-crypto
- Transfers (internal vs external)

## Included

- Documented classification logic
- Implementation guidance:
  - Backend rules, or
  - Structured spreadsheets (early-stage environments)
- Consistent application requirement
- Documented classification methodology

## Excluded

- Reporting output (CARF_REPORTING_OUTPUT)

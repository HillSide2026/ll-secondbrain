# Component Solutions: CARF Program (Minimum Viable but Defensible)

| # | Solution | Description |
|---|----------|-------------|
| 1 | CARF_GOVERNANCE | Named compliance lead; written mandate; escalation rule; policy doc; roles doc |
| 2 | CARF_SCOPING_AND_CLASSIFICATION | Service classification; reportable transaction scope; reportable user scope; classification memorandum |
| 3 | CARF_CLIENT_ONBOARDING | Tax residency self-certification; CARF data collection; onboarding controls |
| 4 | CARF_DUE_DILIGENCE | Re-verification trigger rules; rule table; manual review queue |
| 5 | CARF_TRANSACTION_CLASSIFICATION | Transaction categorization logic; classification rules; documented methodology |
| 6 | CARF_REPORTING_OUTPUT | CRA-facing annual report; structured export; audit traceability |
| 7 | CARF_RECORDKEEPING | Self-certifications; user data changes; transaction records; classification logic retention |
| 8 | CARF_AML_INTEGRATION | AML/CARF data field mapping; separate CARF fields; integration with existing AML profile |
| 9 | CARF_CONTROLS_AND_TESTING | Annual program review; sample testing (10–20 accounts); findings documentation |
| 10 | CARF_DOCUMENTATION | Full documentation set: policy, classification memo, data definitions, reporting methodology, testing log |

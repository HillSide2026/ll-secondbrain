# Procedural Doctrine

Definition: Governance and change-control rules for how doctrine and system rules are reviewed, approved, versioned, deprecated, and migrated.
This section marks the divider before operational execution artifacts (`RULES`, `TESTS`).
Boundary: Does not set substantive operational rules beyond process requirements.

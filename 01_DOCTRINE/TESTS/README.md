# Doctrine Test Fixtures

Definition: Test fixtures and expected outcomes that validate rules and protocol behavior.
Tests are integrity checks for doctrine implementation, analogous to CI for governance.
Boundary: Do not establish doctrine; they verify it.

Typical examples:
- Required metadata is present across doctrine files.
- No rule conflicts with an invariant.
- Policy-to-protocol mapping is complete for enforceable controls.

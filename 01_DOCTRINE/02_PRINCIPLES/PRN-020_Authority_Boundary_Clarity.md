# PRN-020 - Authority-Boundary Clarity

Title: Authority-Boundary Clarity

Statement:
Keep layer roles explicit in interpretation and execution: ML1 decides, ML2 records, System enforces, LL executes. If an action crosses layer scope, route it to the proper authority.

Rationale:
Explicit boundary discipline prevents role confusion, authority creep, and invalid cross-layer actions.

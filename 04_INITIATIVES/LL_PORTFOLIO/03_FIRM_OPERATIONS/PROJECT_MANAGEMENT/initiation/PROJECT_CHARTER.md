# Project Charter

Project ID: LLP-043
Project Path: 03_FIRM_OPERATIONS/PROJECT_MANAGEMENT
Project Type: Management Project
Stage: Initiating

## 1. Purpose
- Govern the project-management tooling and advisory rollup layer for LL so
  project health, stage-gate status, and artifact standards are explicit,
  reviewable, and bounded by ML1 authority.

## 2. Nature of Project
- Management project for project-governance tooling and reporting.
- Advisory and read-only by default.
- Produces rollups, templates, and compliance signals for ML1.
- Does not approve stages, define policy autonomously, or replace project-level
  source artifacts.

## 3. Strategic Rationale
- LL already depends on project-health rollups, artifact templates, and
  governance summaries in its management stack.
- That layer should itself be governed explicitly so advisory tooling does not
  get mistaken for approval authority.
- A clearer project packet also helps separate project management from
  portfolio management and Chief of Staff synthesis.

## 4. High-Level Deliverable
- A defined project-management packet covering scope, authority boundaries,
  artifact standards, rollup outputs, and the conditions under which this layer
  supports ML1 review without acting as ML1.

## 5. Authority
Final approval authority: ML1.

## 6. Promotion Path
- Advance to Planning once ML1 confirms the boundary between project
  management, portfolio management, and repo-level synthesis, and confirms that
  this project remains an advisory tooling layer rather than a policy or
  approval engine.

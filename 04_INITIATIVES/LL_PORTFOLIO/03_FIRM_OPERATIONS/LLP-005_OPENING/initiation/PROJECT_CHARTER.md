# Project Charter

> ML1-approved project packet. Current stage: Executing.

Project: LLP-005 - Opening
Project ID: LLP-005
Project Type: Operational
Stage: Executing

## 1. Purpose

Establish a repeatable opening function that converts engagement-authorized matters into operationally ready open matters.

## 2. Nature of Project

Operational setup workflow.
No doctrine authority.
No legal-delivery decision authority.

## 3. Deliverable

An opening cycle that produces, for each Gate 2 authorized matter:
- opening task execution log
- financial readiness status
- record-status/compliance readiness status
- Gate 3 approval packet
- transition-ready handoff package to LLP-006 maintenance

## 4. Authority

Final approval authority: ML1.
No matter may be treated as Gate 3 complete without ML1-approved opening completion conditions.

## 5. Promotion Path

Upon completion of initiation, planning, and implementation, LLP-005 operates as a standing opening function under `03_FIRM_OPERATIONS` and hands opened matters to LLP-006 Maintenance.

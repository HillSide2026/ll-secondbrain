# Project Charter

Project: LLP-004 - Onboarding
Project ID: LLP-004
Project Type: Operational
Stage: Executing

## 1. Purpose

Establish a repeatable onboarding function that controls the path from qualified lead to signed engagement authorization.

## 2. Nature of Project

Operational gate-control workflow.
No legal-delivery authority.
No doctrine authority.

## 3. Deliverable

An onboarding cycle that produces, for each qualified lead:
- qualification evidence (Gate 1 trigger)
- Clio matter in `Pending`
- engagement agreement drafted and circulated
- Gate 2 authorization packet (signed status + readiness signal)
- exception list for unresolved onboarding blockers

## 4. Authority

Final approval authority: ML1.
No transition to opening without ML1-approved Gate 2 conditions.

## 5. Promotion Path

Executing is authorized. LLP-004 now operates as a standing onboarding
function under `03_FIRM_OPERATIONS` and hands authorized matters to LLP-005
Opening.

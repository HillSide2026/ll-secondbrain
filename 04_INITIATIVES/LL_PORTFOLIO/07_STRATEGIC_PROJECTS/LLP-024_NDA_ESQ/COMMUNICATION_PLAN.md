# Communication Plan

Project ID: LLP-26-24
Project Path: 04_INITIATIVES/LL_PORTFOLIO/07_STRATEGIC_PROJECTS/LLP-024_NDA_ESQ
Stage: Planning

## Purpose

Keep communication limited to what NDA Esq needs to reach a clean
Planning -> Executing decision.

## Required Communication Loops

| Loop | Purpose | Participants | Output |
| --- | --- | --- | --- |
| ML1 decision review | Resolve scope, pricing-boundary, risk, and gate decisions | ML1 + Project Owner | Explicit decisions or escalations reflected in planning artifacts |
| Product/growth/ops working sync | Keep the three workstreams aligned to one launch model | Project Owner + Product Lead + Growth Lead + Operations Lead | Updated assumptions, dependencies, or risks if anything changes |
| Gate submission | Present the final planning packet for implementation decision | ML1 + Project Owner | Updated `APPROVAL_RECORD.md` with the gate result |

## What Must Be Communicated

- any proposed change to MVP scope
- any issue that threatens the 60-day target
- any AI-quality concern that undermines launch confidence
- any CAC, pricing, or channel assumption that looks weak
- any billing, support, or optional lawyer-review control gap
- the final threshold proposal in `METRICS.md`

## Escalation Triggers

- any request to expand into drafting, full-service consultation, or broader contract types
- any blocked dependency that stops product, payment, acquisition, or support readiness
- any milestone slip greater than 5 days
- any AI-quality result materially below the proposed threshold
- any evidence that CAC, revenue, or support assumptions are not credible

## Anti-Bloat Rule

If a communication does not help lock scope, remove a blocker, reduce risk, or
advance the gate decision, it should not exist as a separate planning output.

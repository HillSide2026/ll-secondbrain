# Communication Plan

Project ID: LLP-012
Project Path: 08_MARKETING/LLP-012_FUNNEL2_MANAGEMENT
Stage: Planning

## Review Cadence

| Cadence | Participants | Purpose |
| --- | --- | --- |
| Weekly | ML1 | Review referral progress, qualification quality, page or intake changes, and current conversion signals. |
| Monthly | Portfolio management cycle | Ensure LLP-012 remains aligned with LLP-025, intake management, and portfolio governance outputs. |
| Ad hoc | ML1 + ML2 | Resolve any routing, pricing, or packet-drift issue that materially changes the live F02 surface. |

## Approval Routing

| Artifact Type | Routing |
| --- | --- |
| Offer framing, pricing, and channel sequencing | ML1 approval required |
| Landing-page and intake logic changes | ML1 decision, then implementation support executes |
| Qualification-call script or evidence-schema changes | ML1 approval required before deployment |
| Metrics, threshold, and baseline changes | ML1 review required; record in `METRICS.md` |
| Any material routing change into fulfillment | ML1 approval required with downstream ops visibility |

## Escalation Path

All escalation routes directly to ML1.

Escalate immediately if:

- the qualification gate is being bypassed
- referral surfaces and website surfaces point to materially different offers
- F02 starts generating clearly wrong-fit or low-value work at meaningful volume
- pricing or payment path changes without a recorded ML1 decision
- the metrics packet cannot explain current live performance

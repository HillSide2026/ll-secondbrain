# Assumptions and Constraints

Project ID: LLP-012
Project Path: 08_MARKETING/LLP-012_FUNNEL2_MANAGEMENT
Stage: Planning

## Assumptions

### Market

- Mature Ontario operators with `$1M-$8M` annual cash flow and an active
  accountant relationship are more reachable through trusted intermediaries than
  through undifferentiated search traffic.
- A paid diagnostic framed as a Corporate Health Check is more referable and
  more selective than an open-ended premium consultation.
- Structuring and governance pain is a real buying trigger before litigation or
  crisis, especially when growth, financing, delegation, or documentation drift
  is already visible.

### Operations

- ML1 can deliver the initial Health Check product without a new hire if the
  intake gate filters out low-value and poor-fit matters before consult time is
  allocated.
- GHL and the website stack can support a non-bypass qualification gate and a
  paid booking or purchase path.
- F01 can continue as a bridge channel while F02 is clarified, but F02 should
  become the primary preventative corporate acquisition path over time.

### Conversion

- A mandatory human qualification call increases fit quality for a mid-ticket
  legal service funnel.
- The downstream revenue logic depends on conversion from paid diagnostic to
  remediation work and, where appropriate, fractional counsel or recurring
  relationship work.

## Constraints

### Hard Constraints

| Constraint | Description |
| --- | --- |
| No bypass of the fit gate | F02 must not book or route low-fit operators directly to ML1 without the approved qualification process. |
| Matter value floor | New F02 matters must meet the approved minimum projected matter value of `$1,000`. |
| Practice-area scope | Corporate, Contracts, and Financial Services are all in scope, with primary focus on Corporate and Contracts. |
| Offer discipline | F02 must remain a defined paid diagnostic. It must not collapse into a disguised free-consult model. |
| No F02/F03 conflation | Funnel 02 positioning, landing pages, and intake logic must remain distinct from Funnel 03 regulatory-offer surfaces. |
| ML1 approval on changes | Any change to scope, pricing, channel sequencing, or live-routing logic still requires ML1 approval even though execution has been authorized. |

### Soft Constraints

| Constraint | Description |
| --- | --- |
| Referral network still building | Accountant referral is the adopted primary path, but the relationship base is still being built. |
| Pricing model still needs deployment | The pricing model is now adopted, but Stripe / checkout configuration and live routing still need to reflect the canonical tiers. |
| Qualification-call execution still needs deployment discipline | The call schema is now frozen, but GHL and intake operations still need to enforce it without bypass. |
| SEO is secondary for now | Structuring SEO can support F02, but it should not displace the referral-first architecture. |

## Non-Negotiable Rules

- low-value volume is not success
- F02 must prefer fit quality over throughput
- the qualification call must produce explicit fit evidence, not just a vague
  note that contact occurred
- every live F02 surface should point to one coherent offer path

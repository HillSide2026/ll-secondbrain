# Funnel 02 Proof Architecture

## Purpose
Define proof assets required to convert mid-ticket buyers who need credible structure before purchase.

## Proof Pack (Pre-Purchase)
1. Executive Risk Summary (sample structure)
- Exposure overview sections
- Risk-tier format
- Red/yellow/green matrix example

2. Prioritized Action Plan (sample structure)
- 0-30 day actions
- 90-day actions
- 12-month strategic actions
- Priority and impact scoring example

3. Governance Scorecard (sample structure)
- One-page structural readiness snapshot
- Pillar scoring layout
- Priority flags

4. Anonymized before/after case pattern
- Starting state (structural drift profile)
- Intervention categories
- Post-remediation operating clarity outcomes

5. Turnaround expectation statement
- Scope validation window
- Diagnostic production timeline
- Delivery meeting readiness criteria

## Trust-Transfer Rules
- Show structure and method, not client-specific confidential content.
- Use anonymized patterns, not identifiable client narratives.
- Do not imply guaranteed outcomes.
- Keep proof aligned with actual deliverable scope.

## Required Placement
- Awareness-interest assets: reference proof pack availability.
- Consideration assets: include at least one concrete proof element.
- Inquiry-intake assets: include full proof pack summary link.

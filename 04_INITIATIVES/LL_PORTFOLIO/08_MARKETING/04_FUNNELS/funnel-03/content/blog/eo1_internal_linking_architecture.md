# EO1 Internal Linking Architecture — MSB Cluster

Planning-stage content architecture prepared for EO1 drafts. It is not, by
itself, approval to publish or activate Funnel 03 channels.

## Core Principle

Every page should do four things simultaneously:

- **Anchor up** → to the MSB pillar
- **Branch sideways** → to adjacent edge cases
- **Move down** → to process / registration
- **Pull forward** → to consequence / AML / pressure

If a page only explains itself, it underperforms.

## Linking Rules by Page Type

### Pillar Page
Links to: all classification pages + registration process + operating without registration
Function: capture and distribute traffic into cluster

### Classification Pages (edge cases)
- Link UP → MSB pillar
- Link SIDEWAYS → 2–3 adjacent edge cases
- Link DOWN → registration process + checklist or timeline
- Link FORWARD → operating without registration

### Process Pages
- Link UP → MSB pillar
- Link BACK → 2–3 classification posts
- Link FORWARD → AML obligations + FINTRAC review + consequences

### Consequence Page (Operating Without Registration)
Highest-conversion page.
- Link BACK → pillar + 2–3 classification posts
- Link DOWN → registration process
- Link FORWARD → AML obligations + FINTRAC review
Must show the path forward structurally.

### Compliance Pages (AML + FINTRAC Review)
EO3 bridge.
- Link BACK → registration + consequences
- Link SIDEWAYS → each other
- Link UP → pillar (lightly)

### Operator Page (Flow Mapping)
Authority signal page / hub for serious users.
- Link EVERYWHERE → pillar + facilitator + marketplace + processor + registration

## Anchor Text Strategy

Use search-aligned anchors:

- "money services business in Canada"
- "MSB registration process"
- "payment facilitator vs MSB"
- "operating without MSB registration"
- "ongoing AML obligations"

Avoid: "click here", "learn more", "this page"

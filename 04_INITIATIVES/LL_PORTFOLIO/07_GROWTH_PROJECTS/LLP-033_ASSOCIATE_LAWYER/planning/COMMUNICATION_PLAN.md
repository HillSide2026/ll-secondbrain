# Communication Plan

Project ID: `LLP-033`
Stage: `Planning`

## Communication Rule

This is a strategic planning packet. Communication should stay narrow and
decision-oriented.

## Core Decision Loops

| Loop | Cadence | Purpose | Output |
| --- | --- | --- | --- |
| ML1 planning review | Weekly while planning is open | Review recruitment, supervision, monitoring, and analytics decisions | Updated planning artifacts |
| Economics review | At each material model update | Test whether delegated delivery is still viable | `METRICS.md`, financial-model inputs |
| Supervision and risk review | Before any activation recommendation | Confirm defensibility of review, sign-off, and escalation logic | `RISK_REGISTER.md`, `METRICS.md` |
| Activation-gate review | Once packet is coherent | Decide go / defer / narrow / reject | `../initiation/APPROVAL_RECORD.md` |

## Stakeholder Communication

| Stakeholder | Information Need | Channel |
| --- | --- | --- |
| ML1 | Full decision packet | Direct review of planning artifacts |
| Financial Portfolio / `LLM-003` | Economics assumptions and outputs | Shared planning artifacts and financial-model inputs |
| Risk / supervision stakeholders | Review logic, escalation model, exception criteria | Shared planning artifacts |
| Future role holder | None during planning unless ML1 authorizes live recruiting | Not applicable by default |

## Escalation Triggers

- delegated contribution appears weak or negative in the base case
- supervision model cannot be described clearly enough to defend
- matter-eligibility boundary remains ambiguous
- monitoring framework cannot surface failure early
- planned activation path conflicts with current firm strategy

## Communication Boundary

No external recruiting communication, role advertising, or candidate outreach is
authorized by this planning artifact alone.

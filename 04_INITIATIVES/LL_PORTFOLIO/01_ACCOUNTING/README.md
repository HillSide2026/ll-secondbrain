# 01_ACCOUNTING — Historical Fact

## Purpose

Record what already happened.

## Scope

**Characteristics:** Backward-looking, factual, non-interpretive

### In Scope

- Bookkeeping records
- Historical financial statements
- Reconciliations
- Invoices, payments, expense logs

### Out of Scope

- Forecasting
- Scenario modeling
- Pricing logic
- Recommendations
- Optimization
- Assumptions about the future

## ML1 Authority Statement

ML1 is the sole authority for any interpretation or decision derived from accounting records.

**Rule:** If it could influence a future decision, it DOES NOT belong here.

## Explicit Prohibitions

ML2 must NOT:

- Generate forecasts from accounting data
- Model scenarios using historical figures
- Recommend pricing based on costs
- Optimize based on accounting patterns
- Mix accounting facts with financial models

## Critical Boundary

- **Accounting = facts**
- **Financial Portfolio = models**
- **They must never be mixed.**

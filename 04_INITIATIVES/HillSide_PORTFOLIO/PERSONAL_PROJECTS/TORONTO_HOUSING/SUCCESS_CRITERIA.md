# Success Criteria

## Definition of Success

An explicit Toronto housing direction is chosen and made execution-ready using
written criteria, documented guardrails, and a clear next-step sequence, so the
issue no longer creates unresolved operating drag.

## Key Metrics

| KPI | Definition |
| --- | --- |
| Housing-cost envelope | Defined monthly cost range ML1 is willing to carry for owned Toronto housing |
| Liquidity guardrail | Minimum liquidity or cash-buffer rule that the selected path cannot violate |
| Option-set completeness | Whether ownership in Toronto versus leaving Toronto has been documented and compared |
| Decision status | Whether a concrete direction has been selected, deferred with criteria, or rejected |
| Execution readiness | Whether the selected direction has a realistic next-step sequence |

## Targets

- define the Toronto housing criteria set
- define an explicit monthly cost, liquidity, and flexibility boundary
- formalize the housing budget inside the overall wealth plan
- document the ownership-in-Toronto versus leave-Toronto comparison
- choose a direction and make it execution-ready rather than leaving the matter ambient

## Measurement Method

- written option comparison
- periodic cash-flow check against housing-cost guardrails
- project milestone review

## Timeline for Evaluation

- weekly or biweekly during active decision periods
- milestone-based review until direction is locked

## Initiation Exit Criteria

Initiation should be considered complete when:

- the project purpose, scope, and constraints are documented
- the housing-decision boundary is explicit
- decision criteria and risks are documented
- the project can move cleanly into a planning or execution-ready path

# Worry Eater

A personal project for recording things ML1 is worried about.

## Purpose

Get worries out of your head and into a place where they can be seen clearly,
tracked, and eventually resolved or released. A worry that is written down is
less likely to circulate unexamined.

## Working Artifacts

- `WORRIES.md` — active worry log
- `RESOLVED.md` — worries that are no longer active; resolved, passed, or released

## How to Use

**Adding a worry:** log it in `WORRIES.md` with a date and a short description.
No analysis required at capture — just get it down.

**Reviewing worries:** periodically review `WORRIES.md`. Some worries will have
resolved themselves. Some will need action. Some will need to be accepted and
released. Move resolved or released worries to `RESOLVED.md` with a short note.

**Worry entry format:**

```
| Date | Worry | Project | Domain | Status | Notes |
```

Project: link to the relevant HillSide project ID or name where one exists
(e.g., `HBP-001`, `JAPAN_HOME_DECISION`, `HBP-007`). Use `—` if the worry
does not map to an existing project.

Domains: financial, professional, family, health, legal, housing, other.
Status: active, monitoring, actioned, releasing.

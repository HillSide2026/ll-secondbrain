# Assumptions and Constraints

| Type | Item | Notes |
| --- | --- | --- |
| Assumption | Income is currently being earned | The system is being built around a live cash-flow base rather than a zero-income scenario |
| Assumption | ML1 is willing to change financial habits | Behavior change is required for the system to work |
| Assumption | Long-term commitment exists | The 2030 and 2040 targets require sustained execution over years, not weeks |
| Assumption | Basic banking and investing infrastructure is available | The system assumes access to checking, savings, and investment accounts |
| Assumption | ML1 is willing to define the desired lifestyle explicitly | Freedom and abundance require a real destination, not only numeric accumulation |
| Assumption | Basic-expense and desired-lifestyle expense targets can be separated | Independence and freedom should be measured against different thresholds |
| Assumption | Passive and portfolio income can become a tracked category over time | Later stages depend on asset-driven cash flow, not only net worth |
| Constraint | Time and attention are limited | The system must be lightweight enough to run consistently |
| Constraint | Behavioral consistency is the main execution bottleneck | Complexity must stay low so the system is repeatable |
| Constraint | Existing financial obligations reduce free cash flow | The model must respect current obligations before aggressive investing assumptions |
| Constraint | This project should not become vague self-help journaling | Vision work is in scope only when it changes money decisions and stage progression |
| Constraint | Real ventures belong in `BUSINESS_PROJECTS` | External operating business work is out of scope here |
| Constraint | Stage progression will not be linear | The system may strengthen stability and security long before it reaches independence or freedom |

## Operating Rules to Lock in Planning

- savings and investing happen from defined surplus, not vague leftovers
- basic living expenses and desired lifestyle expenses are tracked separately
- spending should be tested against the stated vision, not only against the budget
- passive / portfolio income coverage should be tracked, not just account balances
- the monthly review cadence should be simple enough to sustain indefinitely
- emergency liquidity is treated as a control layer, not optional cash drift
- asset allocation should favor long-term compounding over reactive tinkering

## Open Planning Questions

- what is the exact current baseline net worth
- what is the current savings rate before any new controls
- what emergency-fund target is appropriate in months of expenses
- what account and contribution order should be treated as default
- what does the ideal day and desired lifestyle actually look like
- what is the monthly basic-expense target versus desired-lifestyle target
- what is the current passive / portfolio income baseline
- what financial-independence number and freedom number should govern the model

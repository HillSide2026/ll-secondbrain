# Dependencies

| Dependency | Type | Relevance | Status |
| --- | --- | --- | --- |
| Written life vision | Directional input | Needed to define what freedom and abundance are supposed to fund | required |
| Income records | Financial data | Needed to fix the current monthly inflow baseline | required |
| Expense records | Financial data | Needed to classify fixed and variable spending accurately | required |
| Asset and liability records | Net-worth data | Needed to establish opening net-worth baseline | required |

| Desired lifestyle expense target | Cost model | Needed to calculate the freedom number | required |

| `HBP-001` / `HBP-002` context | Strategic context | Can inform target path assumptions, liquidity posture, and capital discipline | optional |
| `BUSINESS_PROJECTS` promotion path | Structural | Required if any money engine becomes a formal HillSide business packet | conditional |

## Dependency Notes

- Data quality is the hardest early dependency: a weak baseline will distort every
  later target and review.
- Direction quality matters too: if the desired lifestyle is undefined, the
  freedom and abundance stages cannot be modeled honestly.
- The planning model should not assume perfect categorization on day one, but it
  does require a workable baseline.
- The long-horizon targets can be approved before the full path is modeled, but
  executing should not open until the baseline and contribution logic are
  sufficiently concrete.
- Independence and freedom require separate cost baselines, otherwise later-stage
  qualification will be too vague to govern.

# HillSide Business Project Stage Reference

This file is the canonical stage reference for HillSide business projects.

Only the following project stages should be used for HillSide `HBP` project
tracking:

- `initiating`
- `planning`
- `executing`
- `closing`
- `closed`

## Definitions

- `initiating`: the project is being framed, scoped, and packeted for ML1 stage-gate review
- `planning`: the project has passed initiation and is formalizing scope, roadmap, assumptions, risks, and controls
- `executing`: the project has passed planning and is actively being carried out under approved scope
- `closing`: the main execution path is complete and the project is in close-out, residual handling, or post-closing follow-through
- `closed`: the project is fully closed with no active execution or close-out work remaining

## Rule

Do not use `idea`, `screening`, `validating`, `decision`, `approved`, or
`rejected` as HillSide business-project stage labels.

Those words may appear elsewhere as concept-queue or approval terms, but they
are not authoritative project stages for `HBP-*` records.

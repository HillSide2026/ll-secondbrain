# Project Charter

**Project:** Sale of Granville
**Project ID:** HBP-013
**Project Path:** `04_INITIATIVES/HillSide_PORTFOLIO/BUSINESS_PROJECTS/HBP-013_SALE_OF_GRANVILLE`
**Project Type:** Strategic
**Stage:** Planning

## Sponsor

ML1 (Matthew / HillSide)

## Mission

Build `17409052 Canada Inc` (internal tag: Granville) into a licensed,
market-ready federal MSB asset and eventually divest the licensed entity into
`17513721 Canada Inc`.

## Primary SMART Goal

By `2026-12-31`, `17409052 Canada Inc` has FINTRAC MSB registration in hand or
equivalent license-ready status clearly documented, with a market-ready
licensing and diligence package assembled so the licensed entity can be sold on
an ML1-approved timetable.

## Sale-Readiness Gates

### Minimum Gate

All of the following must be true:

- FINTRAC MSB registration issued; and
- AML program complete enough for buyer review.

### Preferred Gate

Minimum gate satisfied, plus:

- FINTRAC and RPAA status established on the public registry;
- at least one EMI has provided written indication of willingness to onboard 174;
- compliance-software decision made (Rhizome as default unless EMI or bank requires Sumsub);
- Kwiikpay role as a 174 infrastructure path established or ruled out; and
- a buyer can understand the licensed entity's operating support posture without
  treating BaaS software as the asset being sold.

### Stretch Gate

Preferred gate satisfied, plus:

- direct EMI agreement is signed;
- lightweight integration has been tested;
- some actual flow of funds has occurred through the entity;
- compliance workflow is demo-ready;
- diligence folder is complete, including AML program, RPAA analysis / status,
  flow-of-funds documentation with actual transaction evidence, compliance
  software decision, and EMI documentation;
- no exclusivity, lock-in, or dependency that impairs sale; and
- buyer transition steps are mapped.

## Stage Plan

1. Advance RPAA application to remove blockers (domain, safeguarding indication).
2. Secure EMI written indication of willingness to onboard 174.
3. Make compliance-software decision and define the minimum supporting product
   posture.
4. Establish or rule out Kwiikpay as a 174 infrastructure path.
5. Build buyer-readable diligence folder for the licensed entity.
6. Defer active sale outreach until ML1 decides the licensed entity is ready to market.

## Immediate Next Workstreams

- Identify and register a Granville-brand domain; publish single-page site.
- Obtain written indication (LOI-level email) from an EMI.
- Advance Rhizome founder arrangement to a term-sheet conversation.
- Document current RPAA application status.
- Define Kwiikpay role in writing.

## Sale Posture

The current stage is not active go-to-market sale execution.

The current stage is:

- get licensed;
- assemble market-ready posture;
- avoid overbuilding into a live BaaS operating business; and
- preserve the option to sell the licensed entity later on ML1 direction.

## Risks

- FINTRAC timeline: registration delay pushes out the eventual sale window.
- RPAA domain blocker: unresolved domain delays application credibility.
- EMI access: weak supporting infrastructure reduces market-ready credibility.
- Rhizome founder dependency: personal arrangement not yet contracted.
- Overbuild risk: expensive software may be mistaken for the asset rather than
  support for the asset.

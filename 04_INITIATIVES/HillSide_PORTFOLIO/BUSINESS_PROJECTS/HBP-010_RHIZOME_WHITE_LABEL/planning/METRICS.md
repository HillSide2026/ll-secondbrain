# Metrics

Project ID: `HBP-010`
Stage: `Planning`

## Decision-Readiness Metrics

| Metric | Definition | Target | Evidence |
| --- | --- | --- | --- |
| Option set clarity | Number of credible options still under active consideration | narrowed to one preferred option, with at most one fallback | `DECISION_FRAME.md` |
| Control boundary clarity | Ownership of client relationship, branding, support, and delivery is explicit | all four ownership points documented | `DECISION_FRAME.md`, `DEPENDENCIES.md` |
| Economics threshold | Minimum acceptable economics are defined and compared to the preferred path | threshold defined and preferred option assessed against it | `METRICS.md`, `DECISION_FRAME.md` |
| Dependency protection | Exit or control protections are explicit enough to avoid unmanaged lock-in | protections documented or no-go trigger recorded | `DECISION_FRAME.md`, `RISK_REGISTER.md` |
| ML1 gate readiness | Planning packet is complete enough for ML1 decision | all required planning artifacts drafted | planning folder and `../APPROVAL_RECORD.md` |

## Measurement Rule

These metrics are used to determine whether planning has produced a real
decision or only more ambiguity.
